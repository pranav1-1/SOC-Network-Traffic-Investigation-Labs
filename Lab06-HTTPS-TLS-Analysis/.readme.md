# Lab 6 – HTTPS & TLS Traffic Analysis

## Objective

The objective of this lab is to understand how HTTPS secures web communication using the Transport Layer Security (TLS) protocol. In this exercise, Wireshark is used to capture and analyze the TLS handshake, inspect digital certificates, and observe encrypted application data.

This lab demonstrates how SOC analysts investigate encrypted web traffic while identifying useful metadata without decrypting the communication.

---

## Lab Environment

| Device | IP Address |
|---------|------------|
| Kali Linux | 192.168.1.4 |

| Internet | Public HTTPS Websites |

**Operating System:** Kali Linux

**Tool Used:** Wireshark

**Protocol:** HTTPS / TLS

---

## Lab Scenario

An employee reports suspicious activity after visiting a secure website.

Although the traffic is encrypted, the SOC team must verify:

- Which server the user connected to
- Whether the TLS handshake completed successfully
- Which TLS version was negotiated
- Whether the certificate appears legitimate
- Whether encrypted application data is exchanged

The investigation is performed using Wireshark.

---

## Steps Performed

1. Started packet capture in Wireshark.
2. Visited a secure HTTPS website.
3. Applied the following display filter:

```
tls
```

4. Identified the TLS handshake.
5. Examined:
   - Client Hello
   - Server Hello
   - Certificate
   - Encrypted Application Data
6. Compared HTTPS traffic with the HTTP traffic analyzed in Lab 5.

---

## Wireshark Filters Used

```
tls
```

```
tcp.port == 443
```

---

## Key Findings

- HTTPS communication uses TLS encryption.
- The TLS handshake establishes a secure encrypted session.
- Client Hello and Server Hello packets negotiate encryption parameters.
- The server provides a digital certificate for authentication.
- Application data is encrypted and cannot be viewed directly in Wireshark.
- Unlike HTTP, usernames, passwords, and webpage contents are not visible.

---

## SOC Analyst Observations

- Successfully identified a complete TLS handshake.
- Verified encrypted communication over TCP port 443.
- Confirmed the presence of a valid server certificate.
- Observed encrypted application data following the handshake.
- No plaintext credentials or webpage contents were exposed.

## Conclusion

This lab demonstrates how HTTPS protects sensitive information through TLS encryption. While SOC analysts cannot directly view encrypted payloads, they can still investigate handshake information, certificates, protocol versions, and connection metadata to identify suspicious or malicious network activity.
