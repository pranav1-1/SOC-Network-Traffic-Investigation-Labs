# Lab 12 – Telnet Traffic Analysis

## Objective

The objective of this lab is to analyze Telnet network traffic and understand why the protocol is considered insecure. Using Wireshark, a Telnet session was captured between a Kali Linux client and an Ubuntu server to investigate authentication and command execution over an unencrypted connection.

This lab demonstrates how a Security Operations Center (SOC) Analyst investigates legacy protocols and identifies security risks associated with transmitting sensitive information in clear text.

---

# Scenario

ABC Technologies maintains a legacy Linux server that still allows remote administration through Telnet.

The SOC receives an alert indicating that a user logged into the server using Telnet. Since the organization's security policy requires administrators to use SSH, the analyst must determine:

- Which host initiated the connection?
- Which server accepted the connection?
- Which username authenticated?
- What commands were executed?
- Why is this communication considered insecure?
